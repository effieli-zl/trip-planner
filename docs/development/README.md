# Development

Internal engineering docs. Every dev step of meaningful size gets its own detail
doc in this folder, created from [`TEMPLATE.md`](./TEMPLATE.md).

## How dev steps work

1. A step is scoped and listed in the [roadmap](../project-management/roadmap.md).
2. Copy [`TEMPLATE.md`](./TEMPLATE.md) → `step-NN-short-name.md`.
3. Fill in goal, design, tasks, and test/verification notes as you build.
4. Link the new doc from the table below **and** from the [CLAUDE.md](../../CLAUDE.md)
   catalog.
5. On completion, hand off relevant checks to [QA](../qa/test-strategy.md) and
   [UAT](../uat/uat-plan.md).

## Naming

- `step-NN-short-name.md` — `NN` is a zero-padded order (`01`, `02`, …).
- One step = one coherent deliverable that can be built and verified on its own.
